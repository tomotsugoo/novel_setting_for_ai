# NovelSync — AIと作る小説のための設定管理ツール

AIと共同で小説を執筆するとき、複雑なトリックや時系列のねじれ、キャラクターの重複などによって**ハルシネーション（AI の誤った補完）** が頻発します。NovelSync はその課題を正面から解決するための設定管理ツールです。

---

## 解決する課題

### 1. ジャンル固有のハルシネーション
SF・サスペンス・ミステリーでは独自のロジックや世界観ルールが複雑になりがちです。AIはそのルールを文脈から外れて補完しやすい。

**対策:** ジャンル別ルールシートを構造化して管理し、AIへのプロンプトに自動挿入します。

---

### 2. 同姓同名・双子トリックによる混同
登場人物の名前や属性が似ていると、AIは誰のことを指しているか混同します。

**対策:** キャラクターごとに一意のIDと識別フラグ（双子・偽名・変装など）を付与し、同名キャラクターを明示的に区別します。

---

### 3. 非線形な時系列（回想・倒叙・並列時間軸）
物語の語られる順番と実際の出来事の順番がずれると、AIは「どの時点の設定か」を誤ります。

**対策:** 二軸タイムライン管理（物語内時刻 vs 語られる順番）により、各シーンの実時刻と叙述順を分離して管理します。

---

### 4. 読者を欺くトリック・叙述トリック
AIはメタ的に「このキャラクターは実は〇〇だ」という真実を知っているため、読者向けの隠蔽を無視した補完をしやすい。

**対策:** 設定に「公開レベル」を設け、AIに渡す情報と人間（読者）向けの情報を明確に分離します。

---

### 5. 時間経過による設定変化
キャラクターの年齢・関係性・能力・所在地などは物語の進行とともに変化します。AIはある時点の設定を別の時点に誤適用します。

**対策:** 設定値に有効期間（from / to）を持たせ、シーンの実時刻から自動的に正しい設定スナップショットを取得します。

---

### 6. タイムトラベル・ループ構造
複数の時間軸が交差したり、同一人物の別バージョンが存在したりするSFでは、AIは因果関係を容易に壊します。

**対策:** タイムライン分岐を有向グラフで管理し、どのバージョンのどのキャラクターがどの軸に属するかを明示します。

---

## 主な機能

| 機能 | 説明 |
|------|------|
| **キャラクター管理** | 外見・性格・関係性・秘密・変装を構造化。同名キャラを一意IDで区別 |
| **二軸タイムライン** | 物語内時刻と語られる順番を独立管理。シーン別設定スナップショット自動生成 |
| **設定公開レベル** | `世界の真実` / `AI作業用` / `読者向け` の3レベルで情報を分離 |
| **タイムライン分岐グラフ** | タイムトラベル・パラレルワールドを有向グラフで可視化・管理 |
| **プロンプト自動生成** | シーンIDを指定するだけで、そのシーンに必要な設定を圧縮したプロンプトを出力 |
| **矛盾検出** | 設定間の時系列矛盾・同名衝突・因果矛盾をリアルタイム検出 |
| **ジャンルルールシート** | SF・ミステリー・ファンタジーなど、ジャンル固有の世界観ルールを一元管理 |

---

## ターゲットユーザー

- AIと共同執筆を行う小説家・ライター
- 複雑なプロットを持つ作品（叙述トリック・タイムリープ・群像劇）を書く人
- AI補完の精度を高めたいシナリオライター・ゲームデザイナー

## 技術スタック（予定）

| レイヤー | 技術 |
|----------|------|
| フロントエンド | React + TypeScript（管理UI） |
| バックエンド | Node.js / Hono |
| データベース | PostgreSQL（設定データ）+ Redis（キャッシュ） |
| AI連携 | Claude API（プロンプト生成・矛盾検出） |
| グラフ可視化 | React Flow（タイムライン分岐） |
