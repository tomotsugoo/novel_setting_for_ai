# NovelSync — AIと作る小説のための設定管理ツール

AIと共同で小説を執筆するための**ワークフロー基盤**です。ユーザーがプロンプトを書き、AIが設定ツールを参照しながら物語を生成し、満足できるまで設定を調整して再生成を繰り返す——そのサイクルを支えます。

---

## 基本的なワークフロー

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   ① ユーザーがプロンプトを作成                                      │
│      「第3章の冒頭、主人公が駅で犯人と鉢合わせるシーンを書いて」         │
│                        │                                        │
│                        ▼                                        │
│   ② AIが設定ツールから必要な設定を自動取得                           │
│      ・このシーン時点でのキャラクター状態                             │
│      ・その場所・時刻・天候                                         │
│      ・読者にまだ伏せるべき情報の制御                                │
│      ・時系列上の矛盾チェック                                       │
│                        │                                        │
│                        ▼                                        │
│   ③ AIが物語を生成（ハルシネーションなし）                            │
│                        │                                        │
│              ┌─────────┴─────────┐                              │
│              │                   │                              │
│        満足した場合          満足しない場合                          │
│              │                   │                              │
│              ▼                   ▼                              │
│        次のシーンへ      ④ 管理画面で設定を修正                      │
│                           「この時点では犯人はまだ逃走中にする」        │
│                                  │                              │
│                                  ▼                              │
│                          ② へ戻って再生成                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

このループを積み重ねて小説を完成させます。

---

## なぜこのツールが必要か

AIに小説を書かせると、特定のジャンルや構造でハルシネーション（設定の無視・矛盾・混同）が頻発します。原因は「AIに渡す文脈が不正確・不足・過剰」なことです。

NovelSync は**AIに渡す文脈を正確に制御する**ことでこれを解決します。

| 課題 | 原因 | NovelSyncの対策 |
|------|------|----------------|
| SF・ミステリーでの設定無視 | ジャンル固有ルールが文脈から抜ける | ジャンルルールシートを自動挿入 |
| 同姓同名・双子の混同 | AIが同名キャラを区別できない | 一意IDと識別フラグで明示的に分離 |
| 回想・倒叙での時系列混乱 | 語られる順と実時刻が混在する | 二軸タイムライン（実時刻 vs 叙述順）で管理 |
| 叙述トリックの破綻 | AIが「真実」を知っているまま補完する | 設定に公開レベルを設け、AIに渡す情報を制御 |
| 時間経過による設定変化の無視 | AIが古い設定を現在に誤適用する | 設定値に有効期間を持たせ、シーン時刻で自動切替 |
| タイムトラベル・ループ構造の因果破壊 | 複数時間軸をAIが平坦に扱う | タイムライン分岐を有向グラフで管理 |

## 主な機能

### 設定管理（管理画面UI）

**キャラクター管理**
- 外見・性格・能力・秘密・変装・偽名を構造化
- 同名キャラクターを一意IDで区別（双子・クローン・変装対応）
- 関係性マップ（誰が誰を知っているか、いつから知っているか）

**二軸タイムライン**
- 物語内の実時刻と、読者に語られる順番を独立管理
- シーンごとの「その時点での設定スナップショット」を自動生成
- 矛盾（Aが死んだ後に生きている、など）をリアルタイム検出

**設定公開レベル（3段階）**
- `真実` — 世界の実際の状態（叙述トリックの答えを含む）
- `AI作業用` — AIに渡す作業文脈（シーンに必要な情報のみ）
- `読者向け` — 読者が知っている情報（伏線・ミスリードを含む）

**タイムライン分岐グラフ**
- タイムトラベル・パラレルワールド・ループを有向グラフで可視化
- どのバージョンのどのキャラクターがどの軸に属するかを管理

---

### AI連携（生成エンジン）

**プロンプト拡張（自動）**
- ユーザーのプロンプト + シーンIDを受け取る
- 設定ツールから必要な情報を取得してプロンプトに自動付加
- 「このシーン時点での正しい設定」のみをAIに渡す

**矛盾チェック（生成後）**
- 生成されたテキストを設定と照合
- 矛盾が見つかった場合は警告と修正候補を提示

**再生成サポート**
- 何が気に入らなかったかを管理画面で設定変更として記録
- 変更内容を差分として確認してから再生成

## システム構成

```
novel_setting_for_ai/
├── apps/
│   ├── web/                  # 管理画面 UI (React + TypeScript)
│   │   ├── characters/       # キャラクター管理
│   │   ├── timeline/         # タイムライン・分岐グラフ
│   │   ├── settings/         # 設定・公開レベル管理
│   │   └── generate/         # プロンプト入力・生成・再生成UI
│   └── api/                  # バックエンド API (Hono)
│       ├── prompt-builder/   # プロンプト拡張エンジン
│       ├── snapshot/         # シーン別設定スナップショット生成
│       └── conflict/         # 矛盾検出ロジック
├── packages/
│   ├── core/                 # 設定モデル・スキーマ定義
│   └── ai-adapter/           # Claude API連携
└── docs/
    └── schema/               # 設定データのスキーマ仕様
```
