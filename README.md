# NovelSync — AIと作る小説のための設定管理ツール

AIと共同で小説を執筆するための**ワークフロー基盤**です。ユーザーがプロンプトを書き、AIが設定ツールを参照しながら物語を生成し、満足できるまで設定を調整して再生成を繰り返す——そのサイクルを支えます。

---

## 基本的なワークフロー

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   ① ユーザーがプロンプトを作成                                      │
│      「第3章の冒頭、主人公が駅で犯人と鉢合わせるシーンを書いて」         │
│                        │                                        │
│                        ▼                                        │
│   ② Claude が MCP 経由で必要な設定を自動取得                        │
│      ・このシーン時点でのキャラクター状態                             │
│      ・場所・時刻・天候                                            │
│      ・読者にまだ伏せるべき情報の制御                                │
│      ・時系列上の矛盾チェック                                       │
│                        │                                        │
│                        ▼                                        │
│   ③ Claude が物語を生成（ハルシネーションなし）                       │
│                        │                                        │
│              ┌─────────┴─────────┐                              │
│              │                   │                              │
│        満足した場合          満足しない場合                          │
│              │                   │                              │
│              ▼                   ▼                              │
│        次のシーンへ      ④ 管理画面で設定を修正                      │
│                           「この時点では犯人はまだ逃走中にする」        │
│                                  │                              │
│                                  ▼                              │
│                          ② へ戻って再生成                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## システム構成

```
Claude.ai（サブスク）または Claude デスクトップ
        │
        │ MCP Protocol
        ▼
   MCP Server ◀──────────── DB
   (Cloudflare Workers)   (Cloudflare D1)
        ▲                       ▲
        │                       │
        └──────────┬────────────┘
                   │
            管理画面 UI
          (GitHub Pages)
          ユーザーが設定を編集
```

| レイヤー | サービス | 費用 |
|---------|---------|------|
| 管理画面 UI | GitHub Pages | 無料 |
| MCP サーバー | Cloudflare Workers | 無料（10万req/日） |
| データベース | Cloudflare D1（SQLite） | 無料 |

## MCPツール一覧（Claudeが呼び出す関数）

| ツール名 | 役割 |
|---------|------|
| `get_character(id, scene_time)` | 指定時刻時点のキャラクター状態を返す |
| `get_scene_context(scene_id)` | シーンに必要な設定一式を返す |
| `list_characters()` | 登場人物の一覧を返す |
| `check_conflict(description)` | 設定との矛盾を検出する |
| `get_disclosure_level(info_id)` | 読者が知っている情報かどうかを返す |
| `list_timeline_branches()` | タイムライン分岐の一覧を返す |
| `update_scene_progress(scene_id)` | 執筆済みシーンを記録する |

---

## なぜこのツールが必要か

AIに小説を書かせると、特定のジャンルや構造でハルシネーション（設定の無視・矛盾・混同）が頻発します。原因は「AIに渡す文脈が不正確・不足・過剰」なことです。

NovelSync は **AIに渡す文脈を正確に制御する** ことでこれを解決します。

| 課題 | 対策 |
|------|------|
| SF・ミステリーでの設定無視 | ジャンルルールシートを自動挿入 |
| 同姓同名・双子の混同 | 一意IDと識別フラグで明示的に分離 |
| 回想・倒叙での時系列混乱 | 二軸タイムライン（実時刻 vs 叙述順）で管理 |
| 叙述トリックの破綻 | 設定に公開レベルを設け、Claudeに渡す情報を制御 |
| 時間経過による設定変化の無視 | 設定値に有効期間を持たせ、シーン時刻で自動切替 |
| タイムトラベル・ループの因果破壊 | タイムライン分岐を有向グラフで管理 |
